# 心青年咖啡助手 - 网页版部署指南

## 📱 为什么使用网页版？

个人微信小程序有诸多限制，网页版（H5）是最佳选择：

✅ **无需企业认证**  
✅ **可以免费部署**（GitHub Pages、Vercel、Netlify 等）  
✅ **可以直接在手机浏览器中使用**  
✅ **原生支持浏览器的 `speechSynthesis` API，无需云开发**  
✅ **不受微信小程序的审核和权限限制**

## 🚀 快速部署

### 方案一：GitHub Pages（推荐，免费）

1. 将 `coffee helper` 文件夹上传到 GitHub 仓库
2. 进入仓库 Settings -> Pages
3. 选择部署源为 `main` 分支，文件夹为 `/`
4. 点击 Save，等待部署完成
5. 获取访问链接

### 方案二：Vercel（免费，快速）

1. 访问 https://vercel.com
2. 使用 GitHub 账号登录
3. 导入您的仓库
4. 一键部署

### 方案三：Netlify（免费，简单）

1. 访问 https://netlify.com
2. 拖拽 `coffee helper` 文件夹到上传区域
3. 自动部署完成

### 方案四：本地使用（最简单）

直接在电脑或手机浏览器中打开 `index.html` 文件即可使用

## 🎯 使用方法

### 手机浏览器使用

1. 打开手机浏览器（Safari、Chrome、微信内置浏览器等）
2. 输入部署后的网址
3. 点击"直接开始训练"或"开始能力评估"
4. 点击"听语音"按钮，浏览器会自动朗读步骤内容

### 电脑浏览器使用

1. 双击打开 `index.html`
2. 功能与手机版完全一致

## 🔧 功能说明

### 语音播报功能

网页版使用浏览器原生的 Web Speech API，无需任何云服务：

- 自动检测浏览器是否支持语音合成
- 支持多种语音（取决于浏览器）
- 离线也能正常使用

### 数据存储

使用浏览器的 LocalStorage 存储训练记录：

- 数据保存在本地浏览器中
- 刷新页面数据不会丢失
- 清除浏览器缓存会删除数据

## 📝 部署检查清单

- [ ] 确保所有文件在同一个文件夹中
- [ ] 确保 `index.html` 在根目录
- [ ] 测试语音功能是否正常
- [ ] 测试页面切换是否正常
- [ ] 在手机浏览器中测试响应式设计

## 💡 小贴士

1. **建议使用 HTTPS**：部分浏览器要求 HTTPS 才能使用语音功能
2. **添加到主屏幕**：在手机浏览器中可以将网页添加到主屏幕，像 App 一样使用
3. **离线使用**：首次加载后，后续可以离线使用
4. **多设备使用**：在不同设备上使用时，数据不会同步（因为数据保存在本地）

## 🆘 常见问题

**Q: 语音功能不工作？**
A: 确保使用现代浏览器（Chrome、Safari、Edge 等），部分浏览器可能需要授予麦克风权限

**Q: 如何修改老师密码？**
A: 编辑 `index.html` 中 `app.js` 部分的 `teacherPassword` 变量

**Q: 数据会丢失吗？**
A: 只要不清除浏览器缓存，数据会一直保存在本地

## 📞 技术支持

如有问题，请检查：
1. 浏览器控制台是否有错误信息（F12）
2. 浏览器版本是否过旧
3. 是否有网络连接（首次加载需要）
